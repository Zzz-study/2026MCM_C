# 舞蹈比赛粉丝投票分析系统 - 使用指南

## 📋 项目概述

本系统实现了一个完整的舞蹈比赛粉丝投票估计框架，包括：
- ✅ **排名法建模**：整数规划求解粉丝排名（赛季1-2, 28-34）
- ✅ **百分比法建模**：贝叶斯MCMC估计粉丝百分比（赛季3-27）
- ✅ **HMM动态特征**：捕捉选手人气随时间变化
- ✅ **随机森林分析**：特征重要性与预测建模
- ✅ **模型验证**：淘汰一致性、不确定性度量、灵敏度分析

## ❓ 常见问题

### Q1: 编码错误 UnicodeDecodeError

**问题**: CSV文件读取失败

**解决**:
```python
# 方案1: 添加 encoding_errors='ignore'
df = pd.read_csv('file.csv', encoding='utf-8', encoding_errors='ignore')

# 方案2: 尝试其他编码
df = pd.read_csv('file.csv', encoding='gbk')
```

### Q2: 整数规划求解失败（Infeasible）

**原因**: 约束过于严格，无可行解

**解决方案**:
1. 检查数据：judge_rank 和 is_eliminated 是否匹配
2. 放宽约束：允许容差（例如 `>= -0.1`）
3. 检查淘汰选手数量是否异常

### Q3: PyMC3采样不收敛

**现象**: 出现警告 `diverging samples` 或 `low acceptance rate`

**解决**:
```python
# 增加调优次数
trace = pm.sample(1000, tune=1000, target_accept=0.95)

# 降低采样步长
trace = pm.sample(1000, tune=500, target_accept=0.98)
```

### Q4: 内存不足（MemoryError）

**解决**:
1. 分批处理赛季（每次处理5-10个赛季）
2. 减少MCMC采样次数
3. 关闭其他程序释放内存

### Q5: HMM建模失败

**原因**: 参赛周数过少（<3周）

**代码已处理**: 自动跳过，填充默认值1（中人气）

## 🎯 模型验证指标

### 1. 淘汰一致性（Elimination Consistency）

**定义**: 模型预测的淘汰选手与实际淘汰选手是否一致

**计算**:
- 排名法: 总排名（judge_rank + fan_rank）最高者
- 百分比法: 总百分比（judge_percent + fan_percent_mean）最低者

**目标**: ≥ 80%

### 2. 不确定性度量（Uncertainty）

**变异系数（CV）**: 
```
CV = vote_std / fan_votes_mean
```

**解释**:
- CV < 0.1: 低不确定性（高置信）
- 0.1 ≤ CV < 0.3: 中等不确定性
- CV ≥ 0.3: 高不确定性（需谨慎）

### 3. 灵敏度分析（Sensitivity）

**方法**: 改变总票数假设（5000/10000/15000）

**稳定性**: CV越小，模型越稳定
